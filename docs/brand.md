# 북블룸 브랜드 & 확장 전략 v1 (2026-07-10)

> 이 문서는 북블룸을 "앱"에서 "브랜드"로 키우기 위한 기준선입니다. 카피·디자인·기능 결정 시 여기에 맞춥니다.

---

## 1. 브랜드 정체성

### 한 문장
> **"천천히, 나답게 읽는 사람들의 기록장"** — 북블룸(BookBloom)

### 포지셔닝 (경쟁 지형)
- 밀리의서재 = 많이·빨리 읽게 하는 콘텐츠 구독
- 북적북적/플라이북 = 가벼운 인증·소셜
- **북블룸 = 경쟁이 아니라 꾸준함.** "몇 권 읽었나"가 아니라 "오늘도 한 쪽 이어갔다". 톡블룸의 철학("느려도 괜찮아요")을 성인 독서로 옮긴 자리. 이 톤이 유일한 차별점이자 방어선.

### 보이스 (톡블룸 브랜드 가이드 상속)
- 존댓말, 따뜻하고 단정하게. "~해 보세요", "괜찮아요"
- 재촉·죄책감 금지 ("지금 안 읽으면", "뒤처져요" ❌). 3일 쉬면 "다시 이어져요"로 맞이함
- 숫자로 압박하지 않기. 목표 미달도 "천천히 꾸준히면 충분해요"

### 비주얼 시그니처
- 딥 포레스트(#1E3329) + 크림(#F7F4EC) + 소프트그린(#6DBE8A) + 코랄 포인트(화면당 1곳)
- 세리프(Noto Serif KR) = 책의 무드 / Pretendard = 본문
- 🌱 새싹 = 심볼. 완독카드·문장카드·연간결산카드에 항상 "🌱 북블룸" 서명 → 공유될 때마다 브랜드 노출

### 시그니처 자산 (이미 보유 = 브랜드 확산 엔진)
1. **문장 카드** (딥그린+세리프 인용 이미지) — 인스타·스레드 최적
2. **완독 카드** — 완독 순간의 공유 트리거
3. **연간 결산 카드** — 연말 = 브랜딩 골든타임(스포티파이 Wrapped 효과)

## 2. 블룸 생태계 (핵심 확장축)

북블룸은 단독 앱이 아니라 **용디쌤(나용운, 언어재활사 17년)이 만드는 "성장·돌봄 도구" 패밀리**의 일원.

| 앱 | 대상 | 역할 |
|---|---|---|
| TalkBloom | 발달 걱정 부모 | 발달 선별 + 가정 코칭 |
| BabyBloom | 신생아~영유아 부모 | 육아 비서 달력 |
| BloomCare | 양육 가정 | 지원제도 찾기 |
| **BookBloom** | 성인 독서가 | 독서 기록·모임 |

**전략**: 각 앱 설정/푸터에서 서로를 소개(상호 유입). 북블룸 설정의 "블룸 패밀리" 섹션이 그 시작. → 사용자 한 명이 여러 앱을 쓰면 "용디쌤 = 믿을 만한 성장 도구"라는 상위 브랜드가 각인됨. BabyBloom 졸업(37개월+) 유저 → 향후 아이 독서로 BookBloom 연결 가능.

## 3. 콘텐츠 브랜드화 (오픈채팅 × SNS × 블로그)

정기 코너로 "발행 리듬"을 만든다 (불규칙 홍보 < 예측 가능한 코너).

| 코너 | 채널 | 주기 | 자산 |
|---|---|---|---|
| 오늘의 한 문장 | 오픈채팅 | 매일 아침 | 자동화됨(ritual/morning_question.py, 08:31) → 질문카드 |
| 용디쌤의 밑줄 | 인스타/스레드 | 주 1~2회 | 문장 카드 |
| 이달의 책 | 오픈채팅+앱 모임 | 월 1회 | 함께 읽는 책 현황판 |
| 완독 릴레이 | 오픈채팅 | 수시 | 완독 카드 |
| 연간 독서 결산 | 전 채널 | 연 1회(12월) | 연간 결산 카드 |
| 만든 이야기 | 네이버 블로그 | 월 1회 | 스크린샷(현재 초안 발행 대기) |

- 해시태그 세트: #독서기록 #독서앱 #함께읽기 #독서습관 #북스타그램 #책속한줄
- 핵심 원칙: "광고"가 아니라 "용디쌤이 책 이야기하는 계정". 앱은 자연스럽게 딸려감.

## 4. 제품 확장 (리텐션)

- **독서 뱃지** ✅ 구현(v0.10) — 첫 완독·연속 기록·N권·분야 다양성·문장 수집 등. 성취 = 재방문 동기
- 리딩 챌린지(모임): "한 달 2권", "매일 20쪽" 규칙 → 함께 읽는 책과 결합
- 연간 결산의 이벤트화: 12월 자동 알림 + 결산 카드 유도
- 모임장 도구: 이달의 책 지정 알림, 발제 질문 자동(독후 질문 카드 재활용)
- (아키텍처 상 계정 없이 localStorage이므로, 서버 필요한 기능은 모임 테이블(bb_*) 위에 얹기)
