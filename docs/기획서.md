# 북블룸(BookBloom) 서비스 기획서 v1.0

- 작성일: 2026-07-08
- 상태: 초안 (사용자 리뷰 전)
- 서비스명 "북블룸"은 가칭 (Bloom 브랜드 패밀리 연장선 — TalkBloom·BloomCare). 대안: 리드로그(ReadLog), 책갈피, 페이지숲

---

## 1. 한 줄 정의

> **"기록이 쌓이면 습관이 되고, 습관이 모이면 함께 읽게 되는" 성인 독서가를 위한 독서관리 웹 서비스.**

- 타깃: 일반 성인 독서가 (연 5권 이상 읽거나, 읽고 싶은데 흐지부지되는 20~40대)
- 플랫폼: **웹앱(PWA) 먼저** → 반응 검증 후 네이티브 앱 전환
- 확장 전제: 기록·서재 → 습관·통계 → 소셜·챌린지 → 독후활동·교육 순으로 4단계 확장

---

## 2. 문제 정의 (왜 만드는가)

| 사용자의 고통 | 현재 대안의 한계 |
|---|---|
| 읽은 책을 기록하고 싶은데 노션/메모장은 귀찮고 흩어짐 | 범용 도구라 책 검색·표지·메타데이터가 없음 |
| 독서를 습관으로 만들고 싶은데 혼자서는 작심삼일 | 기존 독서앱은 기록 후 "다음 행동"을 안 만들어줌 |
| 읽고 나서 남는 게 없음 (내용을 금방 잊음) | 독후 정리를 도와주는 구조화된 틀이 없음 |
| 같이 읽을 사람이 없음 | 독서모임은 진입장벽(시간·장소·오프라인)이 높음 |

## 3. 경쟁 서비스와 차별화

| 서비스 | 강점 | 빈틈 (우리의 기회) |
|---|---|---|
| 밀리의서재 | 구독 e북 + 통계 | 종이책·타 플랫폼 책은 관리 불가 |
| 북적북적 | 가벼운 기록·인증샷 | 통계·독후활동 얕음 |
| 플라이북 | 소셜·모임 | 개인 기록 도구로는 부족 |
| 알라딘/예스24 마이리브러리 | 구매 연동 | 자사 구매 책 중심, 습관 기능 없음 |
| 노션 템플릿 | 자유도 | 검색·통계·리마인드 전부 수동 |

**차별화 축 3가지**
1. **기록→습관→소셜→활동으로 이어지는 "다음 행동" 설계**: 기록만 하고 끝나는 앱이 아니라, 기록이 스트릭·챌린지·독후활동으로 자연스럽게 연결
2. **구조화된 독후활동**: 책 유형별 독후 질문 카드·필사·서평 틀 제공 (Phase 3, 언어재활사 17년 경력의 "질문 설계" 역량이 차별화 자산)
3. **플랫폼 중립**: 종이책·e북·오디오북·도서관 대출 모두 한 서재에서 관리

## 4. 페르소나

**P1. 습관러 지은 (32, 직장인)** — 주 타깃
- 올해 목표 24권. 1~2월엔 잘 읽다가 흐지부지. 인스타에 가끔 책 사진 올림
- 필요: 읽는 중 책의 진도 관리, 스트릭, 월간 리포트, 가벼운 인증

**P2. 기록러 민수 (41, 다독가)**
- 연 50권+. 노션에 서평 정리하다 지침. 데이터 내보내기·별점·태그에 민감
- 필요: 빠른 책 등록(검색 1초), 상세 기록, 통계, CSV/이미지 내보내기

**P3. 모임러 하영 (28)** — Phase 2 타깃
- 회사 독서모임 총무. 멤버 진도 관리를 카톡+엑셀로 함
- 필요: 방 코드로 초대하는 그룹 챌린지, 진도 현황판, 발제 질문

## 5. 기능 로드맵 (선택한 핵심가치 4개 = 4 Phase)

### Phase 1 — MVP: 기록·서재 + 습관의 씨앗 (핵심)
> 목표: "책 등록이 10초, 오늘 읽었다 체크가 3초"

| 기능 | 상세 |
|---|---|
| 책 검색·등록 | 알라딘 Open API(표지·저자·쪽수 자동). 검색 → 탭 1번으로 서재에 추가 |
| 서재 (3단계 상태) | 읽고싶어요 / 읽는중 / 다읽음. 표지 그리드 뷰 + 리스트 뷰 |
| 진도 기록 | "오늘 ~쪽까지" 입력 → 진행률 바. 완독 시 축하 화면 |
| 한줄평·별점 | 완독 시 별점(0.5 단위)+한줄평 유도. 상세 독서노트는 선택 |
| 독서 스트릭 | 기록한 날 연속 카운트. 홈 화면 최상단 노출 (습관 엔진의 최소 단위) |
| 연간 목표 | "올해 N권" 설정 → 진행률·페이스 표시 ("이 속도면 11월에 달성") |
| 공유 카드 | 완독 기록을 이미지 카드로 저장 → 인스타 공유 (초기 확산 채널) |

### Phase 2 — 습관·통계 심화
- 월간/연간 리포트: 권수·쪽수·장르 분포·최다 별점 작가, 이미지로 공유 가능한 "독서 결산"
- 리마인드: PWA 푸시(또는 이메일) "3일째 기록이 없어요"
- 읽기 기록 캘린더(잔디), 독서 시간대 분석
- 데이터 내보내기(CSV) — P2 페르소나 락인

### Phase 3 — 소셜·챌린지
- **방 코드 기반 그룹 챌린지** (FitMate에서 검증한 UX 재활용): "한 달에 2권", "매일 20쪽" 등 규칙 설정 → 멤버 진도 현황판
- 함께 읽기: 같은 책을 그룹이 읽으며 구간별 코멘트
- 팔로우 없는 가벼운 소셜(빈 방 문제 회피): 친구 서재 구경은 링크 공유 기반

### Phase 4 — 독후활동·교육 (차별화 완성)
- 책 유형별(소설/자기계발/인문) **독후 질문 카드**: 기억 정착용 3질문 + 토론용 3질문
- 필사 모드: 문장 수집 + 필사 카드 이미지 생성
- 독서모임 발제문 자동 초안 (Phase 3 그룹 기능과 결합)
- (선택) AI 요약 대조: 내 한줄평 vs 책 핵심 — 수익화 후보

### 이후 — 앱 전환·수익화
- PWA 지표(재방문율·홈설치율)가 좋으면 네이티브 앱(스토어 출시) 검토
- 수익모델 후보: ① 알라딘 TTB 제휴링크(등록된 책 → 구매 링크, 커미션) ② 프리미엄(무제한 통계·AI 독후활동·데이터 백업) ③ 그룹 챌린지 유료방

## 6. MVP 화면 구성 (Phase 1)

```
[홈]            스트릭 · 읽는중 책 카드(진도 입력 바로가기) · 연간 목표 게이지
[서재]          탭: 읽는중/읽고싶어요/다읽음 · 표지 그리드 · +검색 버튼
[책 상세]       메타데이터 · 진도 히스토리 · 별점/한줄평 · 노트 · 공유카드 생성
[검색]          알라딘 API 검색 → 결과에서 바로 상태 선택해 추가
[기록(통계)]    이번달 권수/쪽수 · 스트릭 캘린더(간이) · 연간 목표
[설정]          목표 설정 · 데이터 내보내기 · 로그인
```
하단 탭 4개: 홈 / 서재 / 기록 / 설정 (모바일 우선 레이아웃)

## 7. 데이터 모델 (초안)

```
users        id, nickname, email, yearly_goal
books        isbn13(PK), title, author, publisher, cover_url, total_pages, category
user_books   id, user_id, isbn13, status(want|reading|done), rating, one_line,
             started_at, finished_at, created_at
progress_logs id, user_book_id, page, logged_date   ← 스트릭·통계의 원천
notes        id, user_book_id, type(note|quote), content, page, created_at
(Phase 3) rooms, room_members, room_rules — FitMate 스키마 참조
```

## 8. 기술 스택 (기존 프로젝트 패턴 준수)

| 항목 | 선택 | 이유 |
|---|---|---|
| 프론트 | Vite + React (talkseed와 동일) 또는 정적 HTML+JS | 확장 전제이므로 React 권장 |
| 백엔드/DB | Supabase (auth + Postgres) | fitmate·talkseed에서 사용 중, 무료 티어 |
| 책 검색 | **알라딘 Open API** (TTB 키 필요, 무료) | 표지 품질·ISBN·제휴 수익까지 연결. 대안: 카카오 책검색 API(키 발급 더 쉬움) |
| 배포 | GitHub Pages (skdyddns-max) | 기존 파이프라인 그대로 |
| PWA | manifest + service worker | 홈 설치·오프라인 서재 열람 |

주의: 알라딘/카카오 API는 CORS 제약이 있을 수 있음 → Supabase Edge Function으로 프록시 (구현 단계에서 확인)

## 9. 성공 지표 (KPI)

| 단계 | 지표 | 기준 |
|---|---|---|
| MVP 검증 | 책 등록 완료율 (검색→추가) | 첫 방문자의 50%+ |
| 습관 | 주 2회 이상 기록 유저 비율 | 30%+ |
| 확산 | 공유카드 생성 수 / 완독 수 | 20%+ |
| Phase 3 진입 조건 | 4주차 재방문율 | 20%+ 일 때 소셜 개발 착수 |

## 10. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 경쟁앱 대비 후발주자 | Phase 4 독후활동(질문 설계)으로 차별화, 초기엔 "가장 빠른 기록"에 집중 |
| 빈 방 문제(소셜) | 소셜을 Phase 3으로 미루고, 링크 공유 기반으로 시작 |

| PWA 푸시의 iOS 제약 | 리마인드는 이메일 병행, 앱 전환 시 해소 |
